\begin{frame}{What is SCIP?}

\textbf{SCIP} --- \emph{Solving Constraint Integer Programs}

\medskip
\begin{itemize}
  \item Fastest non-commercial solver for mixed-integer nonlinear programs (MINLPs)
  \item Also handles satisfiability and constraint integer programs
  \item Supports exact arithmetic for MILPs (mathematical proofs)
  \item Open-source under Apache 2.0 license
  \item Under active development for over 20 years (mostly at Zuse Institute Berlin)
  \item SCIP alumni work on Gurobi, Xpress, COPT, Hexaly, \ldots
\end{itemize}

\begin{frame}{Why do research on optimization?}

Existing solvers are impressive, but they have limits:

\medskip
\begin{itemize}
  \item \textbf{Generic vs.\ specific:} solvers use general-purpose techniques.
  Exploiting problem structure can be orders of magnitude faster
  \item \textbf{Modeling power:} some formulations only become tractable with custom decomposition (e.g., Dantzig-Wolfe, Benders)
  \item \textbf{Beyond MIP:} many real-world problems have side constraints, nonlinearities, or combinatorial substructure that generic solvers cannot exploit effectively
\end{itemize}

\end{frame}

% ---------------------------------------------------------
\begin{frame}{Black-Box vs.\ Open-Source Solvers for Research}

\textbf{Modeling research}
\begin{itemize}
  \item Black-box: compare formulations by solving time, but can't explain \emph{why} one is faster --- lucky heuristic? tighter LP bound?
  \item Open-source: inspect LP bounds at each node, track which cuts are applied, count nodes per subtree --- understand what drives performance
\end{itemize}

\medskip
\textbf{Algorithmic research}
\begin{itemize}
  \item Black-box: can't inspect or modify internal algorithms
  \item Open-source: implement and test new branching rules, cutting planes, propagation, pricing, presolving, all inside the solver
  \item Access conflict analysis, LP warmstarts, and node statistics to evaluate your changes
\end{itemize}

\end{frame}

% ---------------------------------------------------------
\begin{frame}{Plugin-Based Architecture}

SCIP's real strength: a \textbf{modular plugin system} that lets you
customize every aspect of the solving process.

\medskip
\begin{columns}[T]
\begin{column}{0.45\textwidth}
\textbf{Core framework:}
\begin{itemize}
  \item Branch-and-bound tree
  \item LP solver (SoPlex)
  \item Solution \& cut pools
  \item Conflict analysis
  \item Clique table
\end{itemize}
\end{column}
\begin{column}{0.50\textwidth}
\textbf{Plugins are modular:}
\begin{itemize}
  \item Each type has a defined interface
  \item Called by priority order
  \item Multiple plugins per type
  \item Extend in C, C++, or Python
\end{itemize}
\end{column}
\end{columns}

\bigskip
$\Rightarrow$ Not just a solver --- a \textbf{framework} for building solvers
\end{frame}
